# README Standard

## Goal

README should function as an operator-facing entry document, not a loose project note.

## Must include

- what this repository is for
- how the top-level layout is organized
- protected persona/operating-rule files
- env/credential rules
- restart/build commands
- git/GitHub workflow
- Discord/media delivery rules
- where reports/logs go
- current refactor status

## Why

This repository is both a codebase and an operations surface. The README should be useful after a restart, rebuild, or handoff.
