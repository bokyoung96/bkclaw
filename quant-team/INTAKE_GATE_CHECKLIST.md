# INTAKE_GATE_CHECKLIST.md

## Goal

전략 / 백테스트 / 성과 / 탐색 요청에서 **바로 아이디어 제안으로 새지 않도록**
최소 intake gate를 체크리스트로 고정한다.

## Run-Policy Gate (must ask first)

- [ ] iteration / rounds
- [ ] strategies per round
- [ ] reporting cadence
- [ ] stop conditions
- [ ] git/reporting scope when relevant

## Execution Assumptions Gate

- [ ] universe
- [ ] test period
- [ ] benchmark
- [ ] transaction cost / slippage
- [ ] rebalance cadence
- [ ] structure
- [ ] risk constraints

## Rule

위 run-policy gate 확인 전에는:
- 전략 아이디어 확정 제안 금지
- feasibility 설명부터 길게 시작 금지
- 구현 plan 선행 제시 금지

허용되는 첫 응답은 intake 질문 또는 mandate lock 유도까지다.
