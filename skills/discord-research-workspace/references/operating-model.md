# Operating Model

## Phase 1

Start with:
- #main
- #news-flow
- #paper-flow
- #research-lab

Goal:
- Separate intake from synthesis without creating too much overhead.

## Phase 2

Add only if activity justifies it:
- #backtest-lab
- #data-forge

Goal:
- Separate implementation and data work from idea generation.

## Phase 3

Add:
- #archive

Goal:
- Preserve conclusions, lessons, and reusable outputs.

## When to create real OpenClaw agents

Upgrade from channel specialization to real agents only when at least one is true:
- Different channels need different workspaces or memories
- Different routing and permissions are required
- Different operating identities are genuinely useful
- Parallel ongoing work creates enough complexity to justify separation

Otherwise, keep one main agent and use channels plus internal sub-agent thinking.
