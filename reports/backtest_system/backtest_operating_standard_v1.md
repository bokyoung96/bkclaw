# 가재 백테스트 운영 표준안 v1

## 목적
형이 전략/가설/인사이트 탐색을 요청했을 때, 가재가 퀀트DB 기반으로 일관된 백테스트 양식, 실행 절차, 검증 규칙, 보고 형식에 따라 결과를 제공하기 위한 운영 표준.

## 적용 범위
- 퀀트DB 기반 전략 검증
- 데이터 탐색 기반 인사이트 발굴
- 빠른 실험 모드 / 깊은 검증 모드
- dev 채널 진행/검증 알림

---

## 1. 요청 유형

### 1.1 전략 검증형
예: 6개월 모멘텀, 리버설, 퀄리티-밸류 조합, 섹터중립 전략 검증

출력:
- 핵심 지표
- 성과 그래프(이미지)
- 리스크/함정
- 해석

### 1.2 데이터 탐색형
예: 특정 데이터셋에서 이상현상, 예측력, 특징 탐색

출력:
- 탐색 요약
- 후보 가설
- 후속 검증안

### 1.3 전략 발굴형
예: 퀀트DB를 바탕으로 아이디어 제안

출력:
- 전략 후보 리스트
- 구현 가능성
- 필요한 데이터
- 우선순위

---

## 2. 실행 전 확인 양식
백테스트 요청이 오면 바로 실행하지 않고 아래 양식을 먼저 정리한 뒤 형 확인을 받고 진행한다.

- 전략/가설
- 대상 자산/유니버스
- 데이터 소스(현재는 퀀트DB)
- 주기(일봉/주봉/월봉)
- 기간
- 리밸런싱 규칙
- 체결 가정
- 비용/슬리피지
- 제약조건(유동성/시총/거래정지/상장폐지 처리 등)
- 산출물(표/그래프/지표)
- 모드(빠른 실험 / 깊은 검증)

기본 규칙:
- 형이 `백테스트 해줘`라고만 하면 무조건 이 양식부터 보낸다.
- 형이 `간단히`, `빠르게`, `대충`을 쓰면 기본 모드는 빠른 실험 모드.
- 형이 `엄밀하게`, `검증`, `감리`, `실전형`을 쓰면 깊은 검증 모드.

---

## 3. 실행 모드

### 3.1 빠른 실험 모드
목적:
- 1차 감 확인
- 아이디어 러프 체크
- 빠른 의사결정 지원

원칙:
- 최소 검증
- 속도 우선
- 핵심 지표 + 핵심 그래프 중심
- 결과를 빠르게 이미지로 전달

### 3.2 깊은 검증 모드
목적:
- 실전 운용 가능성 점검
- 백테스트 함정 검토
- 리스크 포함 검증

원칙:
- 룩어헤드 검토
- 리밸런싱 로직 검토
- 비용/슬리피지 확인
- 결측/생존자편향/체결가정 점검
- 검증 리포트 포함

---

## 4. 서브에이전트 분업
깊은 검증 모드에서는 필요 시 서브에이전트를 활용한다.

- ALPHA: 전략 가설 정리
- VECTOR: 코드 실행 / 데이터 처리 / 백테스트 구현
- AUDITOR: 룩어헤드, 리밸런싱, 생존자편향, 비용 검증
- RISK: 노출, 회전율, MDD, 군집 리스크 점검
- ARCHIVE: 결과 정리 / 리포트 구조화

운용 규칙:
- 빠른 실험 모드: 서브에이전트 최소화
- 깊은 검증 모드: AUDITOR, RISK 우선 활용
- 최종 답변은 항상 가재가 통합해서 제공

---

## 5. 진행 보고 규칙
작업이 짧으면 바로 결과를 주고, 2~3분 이상 걸릴 것으로 보이면 단계 기반 진행 보고를 한다.

예시:
- 진행 중: 데이터 로드 완료, 전처리 시작
- 진행 중: 시그널 계산 완료, 백테스트 실행 중
- 진행 중: 성과 집계 완료, 그래프 생성 중
- 완료: 결과 요약 전달

주의:
- 너무 잦은 퍼센트 보고는 지양
- 의미 있는 단계별 보고만 사용
- pause 반응성을 위해 긴 poll 대기는 피함

## 6. dev 채널 검증 알림
채널: #dev (`1482514790768447590`)

목적:
- 백테스트 시작/진행/완료 상태 공유
- validation warning/error 알림
- 재현 가능한 실행 흐름 기록

알림 레벨:
- INFO
- WARNING
- ERROR
- DONE

예시:
- INFO: 백테스트 시작
- INFO: 데이터 로드 완료
- WARNING: turnover 급증
- ERROR: 리샘플 주기 오류
- DONE: 결과 리포트 생성 완료

## 7. 표준 산출물
모든 백테스트는 가능하면 아래 산출물을 통일한다.

### 기본 지표
- CAGR
- Annualized Volatility
- Sharpe
- Max Drawdown
- Win Rate
- Turnover

### 기본 그래프
- 누적수익률 / Drawdown
- 월별 수익 분포 또는 heatmap
- 연도별 성과 요약

### 기본 해석
- 전략 요약
- 성과 요약
- 주요 리스크/함정
- 다음 액션

원칙:
- 그래프/시각화는 가능하면 직접 plotting 후 이미지로 전송
- 파일만 던지는 방식 지양

## 8. 코드 구조(초안)
- `src/backtest/` : 백테스트 엔진
- `src/data_sources/` : 퀀트DB 로더
- `src/signals/` : 시그널 생성
- `src/reporting/` : 리포트/그래프 생성
- `src/validation/` : 검증 로직
- `configs/backtests/` : 실행 설정 파일
- `reports/backtests/` : 결과 문서
- `logs/backtests/` : 실행 로그
- `scripts/run_backtest.py` : 표준 실행 진입점
